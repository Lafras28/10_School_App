PHONE DEPLOY CHECKLIST
======================

1) Put the backend online
- Push this project to GitHub.
- In Render: New + -> Web Service -> connect the repo.
- Render will use the included render.yaml file.
- Wait for the service to go live and copy the URL.
- Test it in a browser:
  https://YOUR-RENDER-URL/health

2) Point the mobile app to the live backend
- In the mobile folder, copy `.env.example` to `.env`
- Replace the sample URL with your real Render URL:
  EXPO_PUBLIC_API_URL=https://YOUR-RENDER-URL

3) Build an Android APK
- Open a terminal in the `mobile` folder and run:
  npm install
  npx eas login
  npx eas build:configure
  npx eas build -p android --profile preview

- EAS will give you a download link for the APK.
- Open that link on your Android phone and install it.

4) Quick remote testing without a final APK
- In the `mobile` folder run:
  npm run start:tunnel
- Open the QR/link in Expo Go.

5) Web app with a home-screen icon (iPhone + Android)
- In Render: New + -> Static Site -> connect the same GitHub repo.
- Use these settings:
  Root Directory: mobile
  Build Command: npm install && npx expo export --platform web
  Publish Directory: dist
- Add this environment variable in the static site settings:
  EXPO_PUBLIC_API_URL=https://school-safety-api.onrender.com
- Deploy the static site and open its URL on the phone.
- iPhone: open in Safari -> Share -> Add to Home Screen.
- Android: open in Chrome -> Add to Home Screen / Install App.

Notes
- The app already uses Firebase for most data.
- The hosted backend is still needed for backend fallback calls and PDF export.
- The web app option gives both iPhone and Android users a home-screen icon without needing the app stores.
- Once the APK is installed, the app can work over mobile data or any Wi-Fi, without the laptop being nearby.
